Git#

File name changed, but git is not case sensitive#

It will not be detected by git when change the case of file names.

Use git mv command to update the name, like this:

1git mv data.xml Data.xml

Create .gitignore for existing project while the new ignored file was commited before.#

Steps to remove old commited file or directory from git:

  1. Create a new .gitignore file.
  2. Edit .gitignore to match the file/folder you want to ignore.
  3. Execute the following command: git rm --cached path/to/file or git rm --cached path/to/directory -r. Git will list the files it has deleted. The --cached flag should be used if you want to keep the local copy but remove it from the repository.
  4. Verify that these files are being deleted from version control using git status.
  5. Push the changes to the repository: git push.
